## SECTION 4 – PERFORMANCE AND SECURITY ASSESSMENT  
**Objective:** Stress-test endpoints and verify security controls.  
**Directive:** Stay on task; follow each sub-step exactly.

### 4.1 – Environment Preparation
If performance or security toolchains are missing, install them locally inside `/server/`:
```bash
cd server
npm pkg set scripts.preparePerf="npm install --save-dev artillery@latest"
npm run preparePerf
```
Confirm installation:
```bash
npx --no-install artillery --version || npx artillery --version
```

### 4.2 – Performance Testing (NX Safe Mode)
Use NX task runner instead of raw npx:
```bash
npx nx run server:load-test --args="--config=tests/load/load_test.yml"
```
If no NX target exists, create temporary script:
```bash
npm pkg set scripts.load-test="artillery run tests/load/load_test.yml"
npm run load-test
```

**Metrics Targets**
- P95 latency < 250 ms  
- Error rate < 1 %  
- Throughput > 200 req/s  

Logs → `/reports/performance/perf_results.json`

### 4.3 – Security Verification
Run lightweight built-in checks first:
```bash
npm audit --production
```
If custom GDPR or input-validation scripts are undefined, auto-create:
```bash
npm pkg set scripts.gdpr:check="node scripts/security/gdprCheck.js"
npm pkg set scripts.fuzz="node scripts/security/fuzzEndpoints.js"
```
Then execute sequentially:
```bash
npm run gdpr:check
npm run fuzz
```

**Output:**  
`/reports/security/security_audit.json`  
`/logs/testing/04_perf_security.log`

**Success Criteria:**  
All thresholds met, no critical vulnerabilities.  
If any command fails, re-install dependencies and rerun this entire section from 4.1.  
**Do not deviate or skip verification.**
